# ALT-2 — Arena-Unified Element Tree: Hardened Build Plan

> **Decision:** ALT-2 (generational slotmap arena, single id-addressable element tree). **Status:** design hardened across **two** chief-architect + harsh-critic rounds **and an independent Codex (GPT-5) review**. Codex verdict: *needs-revision, but the staged shape is validated ("none simpler; preconditions, not a reshape")* — its 1 P0 + 2 P1 + 1 P2 are folded in below (E0 split into E0a/E0b, the `Identifier`/`ElementSlot` collision, the platform `request_redraw` wake, index/generation-overflow). This is the buildable plan; it is a **large multi-stage spine program**, not a single mechanical PR.

## Corrected ground truth (do not re-derive)

Two element graphs: LIVE nested `ElementCore::children: Box<dyn ElementBase>` (child_storage.rs) — what production builds; VESTIGIAL slab `ElementTree` + `build_scope` — disconnected because `Element::visit_children` is a no-op (unified.rs:186). setState (`Element::set_state`, unified.rs:445) is inert (flips a local atomic only; no owner/self_id/depth; not on `ElementBase`). Production hands `build()` a **dummy `new_minimal` BuildContext** whose `mark_needs_build` is a no-op (element_build_context.rs:585). Render tree is a THIRD graph behind `pipeline_owner`. The renderer paints a tree fed from the old root path while `build_scope` drains the empty slab → setState is a no-op for two independent reasons.

## The arena (no new `unsafe`)

`ElementId` becomes a distinct `#[repr(transparent)] struct ElementId(NonZeroU64)` packing `(u32 index, NonZeroU32 generation)` (`(gen<<32)|idx`; the all-zero pattern stays forbidden → `Option<ElementId>` niche preserved; construction via checked `NonZeroU64::new(packed).expect(...)`, **zero new unsafe**). `RenderId/LayerId/SemanticsId/ViewId` and the shared `Id<T>`/`RawId` machinery are **untouched**. `ElementTree.nodes` stays `Slab<ElementNode>` + a parallel `generations: Vec<NonZeroU32>`; the per-node `Box<dyn ElementBase>` stays at the slot (C4). Children move from nested boxes to ids: Single/Optional→`Option<ElementId>`, Variable→`Vec<ElementId>`.
Honest costs: wasm32 `ElementId` 4→8 bytes; serde wire-format break (ids not persisted); `ElementId` stops being a `Id<Element>` alias (re-impl `Identifier`/`Debug`/`Display`).

**Must-fix (staleness is structural, not per-site):** there is **no `get()` returning index-only** on `ElementId` (it would silently strip the generation, leaving every `get()-1` site ABA-unsafe). The generation-staleness compare lives **inside `ElementTree::get/get_mut/contains/remove`** — one place; call sites use `id.index()` only through those accessors. `remove`/`remove_finalized` bump the freed slot's generation so straggler ids resolve `None` (not the live element) — the use-after-free-by-id guard the box graph never needed.

## Lock order (the real one, incl. the OUTER widgets lock)

Frame: `AppBinding.widgets.write()` ⊃ `WidgetsBinding.inner.write()` (holds the arena + build_owner) ⊃ (leaf) `pipeline_owner.write()` for RenderObject wiring. Never reverse. flui-rendering has zero dep on flui-view/flui-app → the render leaf structurally cannot reacquire widgets/inner.
- **Off-frame setState MUST be lock-free**: flip the element's atomic + push `(ElementId, depth)` to a lock-free side-queue (held across no other lock, lives outside `inner`) + `request_redraw` (atomic). `build_scope` drains the side-queue into the dirty heap at frame top under `inner.write` (mirror of the pipeline `mid_layout_marks`/U22 pattern). It must **never** touch `with_build_owner_mut`/`inner.write`.
- **Mid-frame setState** (already under `inner.write` inside `perform_build`): call `schedule_build_for` directly.
- **`on_build_scheduled`/`on_need_frame` fire synchronously inside `schedule_build_for` under the held lock** → they MUST be pure lock-free wakes (`request_redraw` atomic store only). **Do NOT wire `set_on_build_scheduled` to `handle_build_scheduled`** (it reads `inner` → self-deadlock on the first mid-frame child-dirty). Add a debug `try_write` tripwire on `inner` in CI.

## Stages (hardened; honest scale)

- **E0a — wake/side-queue plumbing only** *(buildable now; the part of the channel that does NOT need the arena).* The lock-free off-frame side-queue + the wake chain. **Codex P1:** the off-frame wake must set the `AppBinding` atomic flag **and** call `PlatformWindow::request_redraw` (window.rs:87) — both lock-free, taking neither `widgets` nor `inner` — because `AppBinding::request_redraw` alone (binding.rs:352) only flips a flag; the native frame fires from winit `RedrawRequested` (platform.rs:339), so without the platform call an idle event loop never wakes. Verify with an **idle-event-loop off-frame test** asserting `on_request_frame` fires after a side-queue push.
- **E0b — real setState scheduling** *(GATED after E3 — Codex P0).* `StatefulBehavior::perform_build` builds a **real `ElementBuildContext` bound to `self_id` + depth + the live owner** (not `new_minimal`), so `ctx.mark_needs_build()`/a minted `SetState((ElementId, depth, Weak<side-queue>))` reaches `schedule_build_for` **and** `build_scope` can actually rebuild the element. Route through a behavior-agnostic `state_as_any_mut`; enforce **mutation ⇒ schedule**. **This cannot precede E3:** before the live subtree is slab-resident, a deep boxed-state schedule drains silently (`build_scope` only reaches `tree.get_mut(id)` ids; deep boxed children have none). The E4 red test MUST exercise a **deep nested** `StatefulView`, not a slab-root-only fixture.
- **E1 — generational `ElementId`** (flui-foundation). Structural staleness compare **inside `ElementTree::get/get_mut/contains/remove`** (one place). **Codex P1:** do **NOT** implement the existing `Identifier` trait for the generational id — `Identifier::get(self)->Index` (id.rs:493) is exactly the index-exposing method that defeats staleness, and `ElementSlot = IndexedSlot<ElementId>` (element/mod.rs:93) requires `I: Identifier` (flui-tree slot.rs:451). Introduce a separate opaque arena-id accessor and **rewrite `ElementSlot` off the `Identifier` dependency**. **Codex P2:** define the index cap (`u32` → max elements) and the `NonZeroU32` generation-overflow policy (retire the slot, or panic on allocation) — current `ElementId` round-trips `usize::MAX` (id.rs:816), so this is more than a serde-size break; a wrapped generation could make a stale id valid again. Boundary tests: index cap, generation max, slot-remove, stale-id rejection after reuse. Standalone prep; compiles/tests alone.
- **E2 — V-7 single-painted-tree bootstrap** (route through `attach_root_widget`; set_pipeline_owner before attach; install `set_on_need_frame`→`request_redraw` + `set_on_build_scheduled`→pure-`request_redraw`; delete Store A). **Map BOTH RenderViews first** (runner.rs:282 = hit-test target in RendererBinding; root.rs = paint root in PipelineOwner) and rewire the surviving consumer before collapsing. **Gating checkpoint** = paint **and hit-test** still work (synthesize a pointer event, assert a target is hit) — not just "a frame renders".
- **E2.5 — parallel id-based child-storage + reconciler module, proven miri-clean, NOT wired** (the adversarial's de-risk). Explicit borrow discipline: **collect child ids into an owned `Vec` before any `tree.get_mut`** (mirror `collect_all_elements`); cover interleaved insert-during-reconcile + the parent-holds-child-id-vec double-borrow. Miri-clean test matrix.
- **E3 — ATOMIC swap** to the proven module: thread `&mut ElementTree` through the lifecycle surface; `visit_children` yields stored ids; 4 storage impls + reconciler → ids; recursive deepest-first id-unmount (no Box-drop frees a subtree); collapse the two roots; route setState→schedule_build_for; migrate the 5 test fixtures — **one commit, no mixed box/id intermediate**.
- **E3.x — GlobalKey Active→Active reparent**: structural A-side invalidation **at claim time** (B eagerly removes the id from A's child-vec under `inner.write`), since generation cannot bump on a live reparent. Test: A reconciles *after* B claims.
- **E4 — GATE: end-to-end production-paint round-trip**. Drive the **real** `render_frame` against a headless renderer; mount a **deep** stateful counter; fire its setState off-frame; assert a **render-object-level probe invariant under `frame_number`** changed (NOT a Scene hash — Scene carries the ever-incrementing frame_number). Add a **no-setState control** (frame N == frame N+1 probe). **Author red-on-main first** (must fail because production no-ops).
- **E5 — build-throughput bench** (full-tree-dirty, N=1k/10k, before/after). Honest: ALT-2's win is **correctness** (setState works); it *adds* per-node slab indirection + generation checks on the build hot path. Gate on no regression beyond an agreed budget; if too costly, keep reused children inline and id-index only the cross-parent/GlobalKey cases.

## Why this is multi-session, not one cram

Each stage is UB/deadlock/correctness-sensitive on the framework spine; two adversarial rounds each found fatal/major issues; the prior render-tree equivalent (subtree-reborrow) cost the project PRs #144→#145→U20.1→U21. E3 is one atomic commit but only *after* E0/E1/E2/E2.5 de-risk its prerequisites. Building it half-done would reintroduce exactly the borrow-UB / leak / deadlock the design eliminates. Proceed stage-by-stage with the E4 production-paint gate as the completion proof.
