# Input + Frame-Loop Repair — `flui-interaction` × `flui-scheduler`

## Summary

Single comprehensive PR repairing the FLUI input + frame-loop layer. Closes 44 findings from [`docs/research/2026-05-21-flui-interaction-scheduler-audit.md`](../research/2026-05-21-flui-interaction-scheduler-audit.md) in atomic-commit-per-unit shape matching [PR #84](https://github.com/vanyastaff/flui/pull/84) precedent. ## Problem Frame

The audit at [`docs/research/2026-05-21-flui-interaction-scheduler-audit.md`](../research/2026-05-21-flui-interaction-scheduler-audit.md) (44 findings, 8 CRITICAL / 21 HIGH / 13 MED / 5 LOW across 50 source files + 21.4K LOC) surfaces three layers of brokenness compounding across `flui-interaction` + `flui-scheduler`:

1. **The most-used user-facing input API is silently leaking.** `Ticker` has no `dispose()`, no `Drop`, no `disposed: AtomicBool` — Flutter's [`ticker.dart:362-379`](../../.flutter/flutter-master/packages/flutter/lib/src/scheduler/ticker.dart) is `@mustCallSuper` that cancels the pending `TickerFuture` + leaves a use-after-dispose marker. FLUI's `Ticker::start` silently overwrites prior state instead of throwing the explicit `FlutterError('A ticker was started twice.')` at [`ticker.dart:188`](../../.flutter/flutter-master/packages/flutter/lib/src/scheduler/ticker.dart). PR #84 already added this exact pattern (dispose + disposed-state assertion) to `ChangeNotifier` at [`flui-foundation/src/notifier.rs`](../../crates/flui-foundation/src/notifier.rs) — Ticker is the obvious next adoption site. Same pattern absent across `Recognizer::dispose` (skips arena reject + router unregister per Flutter [`recognizer.dart:485-493`](../../.flutter/flutter-master/packages/flutter/lib/src/gestures/recognizer.dart)), `FocusManager`, and `GestureBinding::hit_tests` (DashMap leak on device disconnect mid-down).

2. **Parallel-implementation clusters defeat the Behavior-loyal philosophy.** Three `Ticker` shapes (`Ticker` + `ScheduledTicker` + `TypestateTicker<S>`) where Flutter has ONE — `ScheduledTicker::tick_and_reschedule` re-locks the inner Mutex 3× per tick + allocates a fresh `Box<dyn FnOnce>` each frame, the **single largest per-frame allocation hot path in the scheduler crate**. `TickerProvider::schedule_tick(callback)` (one-shot callback) misshapes Flutter's `TickerProvider.createTicker(callback) -> Ticker` factory at [`ticker.dart:248`](../../.flutter/flutter-master/packages/flutter/lib/src/scheduler/ticker.dart) — blocks any `flui-animation::SingleTickerProviderStateMixin` equivalent landing. Two `GestureRecognizerState` types (a container struct at [`recognizer.rs:59`](../../crates/flui-interaction/src/recognizers/recognizer.rs) + a state enum at [`primary_pointer.rs:60`](../../crates/flui-interaction/src/recognizers/primary_pointer.rs)) re-exported under three names (`PrimaryPointerState`/`PrimaryPointerStateHelper`/`GestureRecognizerState`) at [`recognizers/mod.rs:74-79`](../../crates/flui-interaction/src/recognizers/mod.rs) — concrete recognizers bypass both with ad-hoc enums, a third `GestureState` is dead. `OneSequenceGestureRecognizer` + `PrimaryPointerGestureRecognizer` traits + helpers (~823 LOC) have **zero `impl ... for` blocks** workspace-wide where Flutter [`recognizer.dart:404,611`](../../.flutter/flutter-master/packages/flutter/lib/src/gestures/recognizer.dart) uses them as the canonical base for all 7 recognizers we mirror. Two `FocusManager` implementations live in parallel — `focus.rs::FocusManager` (flat-state, used by `global()`) vs `focus_scope.rs::FocusManagerInner` (tree-based, internal). `FocusManager::focus_next` is `tracing::warn!("not yet implemented")` while `FocusScopeNode::focus_next_in_scope` is fully implemented + unreachable — **Tab navigation, the single most-used keyboard API in any UI framework, is a log line**.

3. **Constitution principle violations + zombie surface bloat hide in plain sight.** Six `from_u8` panic sites in [`scheduler.rs:496,501,1035,1065,1111,1140`](../../crates/flui-scheduler/src/scheduler.rs) called via raw atomic loads in production paths violate Principle 6 ("No `unwrap()`/`println!`/`dbg!`" + `unimplemented!()` same class). `VsyncScheduler::new(0)` [`vsync.rs:167`](../../crates/flui-scheduler/src/vsync.rs), `FrameDuration::from_fps(0)` [`duration.rs:513`](../../crates/flui-scheduler/src/duration.rs), `set_time_dilation(non-positive)` [`config.rs:97`](../../crates/flui-scheduler/src/config.rs) all `assert!`-panic. ~3,995 LOC of zombie surface: `typestate.rs` (232 LOC, 13 ZST markers + 4 marker traits, 0 impls), `OneSequence` + `PrimaryPointer` (823 LOC, 0 impls — the very traits Flutter's parallel are), `TypestateTicker<Idle/Active/Muted/Stopped>` (392 LOC, 0 consumers), `prelude_advanced` + `Handle<M>` + 4 ZST priorities + `TypedTask<P>` + 3 ext traits + `ToMs/ToSecs` (~650 LOC, 0 external consumers), `Scheduler::arc_instance()` (parallel singleton, 0 callers, silent dual-state hazard), `VsyncDrivenScheduler` (134 LOC, 0 production consumers), `PointerEventData` + `PointerEventKind` + `make_pointer_event` compat layer (~150 LOC, used only by `testing/`), `OrderedTraversalPolicy` + `DirectionalFocusPolicy` (~120 LOC, never assigned). `testing/` submodule (1,099 LOC) ships in release binaries — no feature gate. `Microseconds(i64)` allows negative values where every production constructor uses `u128 as i64`. Two `FocusManager` types, two `IndexedSlot` types (interaction defines its own duplicating `flui-tree::IndexedSlot` — collision-risk repeat of PR #84's R2), `PointerId` re-defined locally where `ui_events::pointer::PointerId` is already imported in 14+ files.

Adjacent surface: persistent/post-frame callbacks in [`flui-scheduler/src/scheduler.rs`](../../crates/flui-scheduler/src/scheduler.rs) currently support `remove_persistent_frame_callback` + `cancel_post_frame_callback` returning CallbackIds — diverges from Flutter strict ["Persistent frame callbacks cannot be unregistered"](../../.flutter/flutter-master/packages/flutter/lib/src/scheduler/binding.dart) at `binding.dart:773` + ["Post-frame callbacks ... called exactly once"](../../.flutter/flutter-master/packages/flutter/lib/src/scheduler/binding.dart) at `:802`. Lifecycle state changes ([Flutter `binding.dart:414-441`](../../.flutter/flutter-master/packages/flutter/lib/src/scheduler/binding.dart)) auto-update binding state in Flutter — FLUI has parallel `lifecycle_state_listeners` registry on Scheduler where Flutter routes listeners through `WidgetsBindingObserver.didChangeAppLifecycleState` at the widgets binding layer. `Priority` is a closed 4-variant FLUI enum (`Idle/Build/Animation/UserInput`) where Flutter has 3 open-class values with operator overloading (`idle=0, animation=100000, touch=200000` + `operator+(int offset)`) — Flutter's offset arithmetic has zero usages in Flutter framework code itself (grep empty), but the numeric values diverge (FLUI's `Animation` is implicit 2, Flutter's is 100000).

The cost is invisible because no test exercises the broken paths. FLUI advertises Flutter widget-tree parity and 60fps animation, but **no `flui-animation` Ticker can dispose cleanly**, **no Tab key navigates focus**, **no Recognizer cleanly disposes mid-gesture**, **no `PointerEvent` re-dispatch path stays under 2+N+M `RwLock::read()` per event**. Repeat reviews of the same broken surface across distinct PRs would be the pattern the user explicitly forbids per [memory `no-quick-wins-vanyastaff`](../../../../.claude/projects/C--Users-vanya-RustroverProjects-flui/memory/no-quick-wins-vanyastaff.md) — bundle into one repair pass matching PR #84 precedent.

## Actors

- **A1. Gesture-consuming view author.** Wires `View::build` to construct a recognizer (`TapGestureRecognizer::new()`, etc.), attaches it via `View::on_pointer_event` (today) or a future `RecognizerView` widget. Expects `recognizer.dispose()` to clean up arena/router state and `recognizer.on_tap_down/on_tap_up` to fire in Flutter-faithful order.
- **A2. Animation author (`flui-animation` consumer, currently disabled).** Constructs `Ticker` via `TickerProvider::createTicker(callback)`, expects `start()` to return `TickerFuture`, expects `dispose()` to cancel the future and panic on use-after-dispose. Today gets `schedule_tick(callback)` instead of a `Ticker` factory.
- **A3. Focus-aware widget user.** Holds focus state in a `FocusScopeNode`, expects `FocusManager::focus_next()` to traverse the focus tree per Tab key. Today gets `tracing::warn!("not yet implemented")`.
- **A4. Frame-callback registrant.** Calls `Scheduler::add_persistent_frame_callback(callback)` or `add_post_frame_callback(callback)` expecting Flutter-strict semantics (persistent runs every frame, post-frame runs once at frame end). Today FLUI offers a `remove_persistent_frame_callback` API that's a foot-gun.
- **A5. Pointer-event router consumer.** Registers handlers via `PointerRouter::add_route(pointer_id, handler)` expecting Flutter-O(1) lookup + per-pointer-then-global dispatch order. Today gets 2+N+M `RwLock::read()` per event + reversed dispatch.
- **A6. Platform-event ingest (`flui-platform`).** Pushes `ui_events::pointer::PointerEvent` into `GestureBinding::handle_pointer_event`. Expects pointer lifecycle (`Down → Move → Up`/`Cancel`) to drain HashMap entries on disconnect. Today entries leak.
- **A7. Application lifecycle owner (`flui-app`).** Receives `AppLifecycleState` transitions from platform, expects scheduler `frames_enabled` to flip per Flutter [`binding.dart:414-441`](../../.flutter/flutter-master/packages/flutter/lib/src/scheduler/binding.dart). Today must manually toggle.

## Key Flows

- **F1. Ticker lifecycle.** A2 calls `let mut ticker = provider.create_ticker(|elapsed| { ... });` then `ticker.start();`. Ticker's `_state` transitions `Inactive → Active`, `TickerFuture` created. Scheduler dispatches `tick` callbacks every frame. A2 calls `ticker.dispose()`; `TickerFuture` cancels, `disposed = true` atomic-set, `Drop` is a no-op now (dispose-already-fired). Subsequent `start/stop/mute/unmute` calls `debug_assert!(!self.disposed, "use after dispose")` in debug; `tracing::warn!` + no-op in release. Flutter parity: [`ticker.dart:185-208,362-379`](../../.flutter/flutter-master/packages/flutter/lib/src/scheduler/ticker.dart). Covered by **R3, R4, R5, AE1, AE2**.

- **F2. Recognizer lifecycle + arena cleanup.** A1 attaches `TapGestureRecognizer`. Pointer-down → `recognizer.add_pointer(event)` enters arena. Pointer-up before timeout → `recognizer.handle_event(...)` resolves arena → `on_tap` fires. A1 then drops the view; `recognizer.dispose()` MUST (a) reject any open arena entries via `arena.resolve(pointer_id, GestureDisposition::Rejected)`, (b) unregister all `PointerRouter::add_route` calls for tracked pointers, (c) clear timer service entries, (d) set `disposed = true`. Today: dispose skips (a) + (b) + (c). Flutter parity: [`recognizer.dart:485-493`](../../.flutter/flutter-master/packages/flutter/lib/src/gestures/recognizer.dart). Covered by **R8, R9, R10, AE5, AE6**.

- **F3. Focus traversal.** A3 calls `FocusManager::global().focus_next()`. Manager walks active `FocusScopeNode` tree via `FocusScopeNode::focus_next_in_scope(current_focus)` — already-implemented at [`focus_scope.rs:663`](../../crates/flui-interaction/src/routing/focus_scope.rs). Result: `Option<FocusNodeId>` of next focusable. Manager calls `request_focus(next_id)`. Today: warn + no-op. Covered by **R6, R7, AE3, AE4**.

- **F4. PointerEvent route + dispatch.** A6 emits `PointerEvent::Down` → `GestureBinding::handle_pointer_event` → hit-test → `PointerRouter::route(event)`. Router consults `routes: HashMap<PointerId, Vec<RouteHandler>>` O(1), dispatches per-pointer handlers first (Flutter order at [`pointer_router.dart:124`](../../.flutter/flutter-master/packages/flutter/lib/src/gestures/pointer_router.dart)), then global handlers. Pointer-cancel/up → handlers drain HashMap entry. Today: routes are `RwLock<HashMap>` taken 2+N+M times per event with linear `Arc::ptr_eq` re-check; dispatch is global-first; HashMap entries leak on device-cancel-mid-down. Covered by **R11, R12, R13, AE7, AE8**.

- **F5. Frame callback dispatch with Flutter-strict immutability.** A4 calls `Scheduler::add_persistent_frame_callback(|frame_time| { ... })`. Returns `()` (no handle, no removal). Callback fires every `handle_draw_frame`. A4 also calls `add_post_frame_callback(|frame_time| { ... })` — fires exactly once at end of next `handle_draw_frame`, then dropped. Today: both return `CallbackId` + support removal — divergence from Flutter strict [`binding.dart:773,802`](../../.flutter/flutter-master/packages/flutter/lib/src/scheduler/binding.dart). Covered by **R14, AE9**.

- **F6. Lifecycle state → scheduler frames_enabled.** A7's platform layer emits `AppLifecycleState::Paused`. `Scheduler::handle_app_lifecycle_changed(state)` auto-flips `frames_enabled: AtomicBool` based on state (`Resumed` → true, `Inactive`/`Paused`/`Hidden`/`Detached` → false). Tickers receive mute via lifecycle observer. Flutter parity: [`binding.dart:414-441`](../../.flutter/flutter-master/packages/flutter/lib/src/scheduler/binding.dart). Covered by **R15, AE10**.

## Key Decisions

- **Bundle all 44 audit findings + 8 open-question resolutions + flui-animation source migration into ONE PR.** Per user directive ("мы ничего не пропускаем и делаем сразу хорошо") + [memory `no-quick-wins-vanyastaff`](../../../../.claude/projects/C--Users-vanya-RustroverProjects-flui/memory/no-quick-wins-vanyastaff.md). The 44 findings share dispose-pattern / typestate-cleanup / parallel-type-consolidation infrastructure; splitting forces double passes through same lifecycle code. Cross-crate ripples bounded.
- **Zero-dependency deletions (Wave 1) sequenced FIRST.** ~3,995 LOC of pure zombie deletion has zero consumer impact, gets out of the way before consolidation. Reduces blast radius of Wave 4's larger restructures (recognizer migration through cleaner surface).
- **Type-collision + Principle 6 (Wave 2) sequenced SECOND.** Enum canonicalization (R3) blocks recognizer migration in R14 — recognizers need one FSM enum to target. `TryFrom<u8>` (R4) + `Result<_, InvalidValue>` constructors (R5) + `u64` Microseconds (R6) + `ui_events::pointer::PointerId` widening (R7) all establish the type baseline before Wave 3 dispose plumbing references them.
- **Wave 3 dispose pattern applied UNIFORMLY across `Ticker` + `Recognizer` + `FocusManager` + `GestureBinding` in adjacent commits.** Matches PR #84's `ChangeNotifier::dispose` adoption pattern; reviewer can compare across all sites in one PR diff. Per-type commits, not per-method, to keep dispose semantics atomic per consumer.
- **Recognizer migration (Wave 4 R14) one-atomic-commit-per-recognizer.** Seven recognizers × ~150-250 LOC each. Per Flutter base class mapping: Tap+LongPress → PrimaryPointerGestureRecognizer; Drag+Scale+ForcePress → OneSequenceGestureRecognizer; DoubleTap+MultiTap → GestureRecognizer direct. Confirmed via [`tap.dart:202`](../../.flutter/flutter-master/packages/flutter/lib/src/gestures/tap.dart) + [`long_press.dart:262`](../../.flutter/flutter-master/packages/flutter/lib/src/gestures/long_press.dart) + [`monodrag.dart:81`](../../.flutter/flutter-master/packages/flutter/lib/src/gestures/monodrag.dart) + [`scale.dart:345`](../../.flutter/flutter-master/packages/flutter/lib/src/gestures/scale.dart) + [`force_press.dart:117`](../../.flutter/flutter-master/packages/flutter/lib/src/gestures/force_press.dart) + [`multitap.dart`](../../.flutter/flutter-master/packages/flutter/lib/src/gestures/multitap.dart) (DoubleTap+MultiTap base on `GestureRecognizer`).
- **`PointerId` widening (R7) bundled into Wave 2 as one atomic commit despite ~20 site ripple.** Mechanical sweep, all under `flui-interaction`. Splitting per-site adds 20 commits for no review benefit.
- **`testing/` feature-gate (R21) lands with `default = []`.** Zero workspace consumers confirmed; no migration needed. External `dev-dependencies` users opt in via `features = ["testing"]`.
- **`flui-animation` migration (R26) lands in this PR despite crate being disabled.** `controller.rs:8` imports `ScheduledTicker` — R13 absorbs that type. If migration deferred, re-enabling `flui-animation` later breaks compile. Cheap migration now is the no-quick-wins answer.
- **`Priority` numeric realignment (R25) without operator+/- offset arithmetic.** Flutter's offset arithmetic at [`priority.dart:40,53`](../../.flutter/flutter-master/packages/flutter/lib/src/scheduler/priority.dart) has zero usages in Flutter framework code itself (verified via grep). Closed enum stays Rust-simpler. Numeric realignment preserves forward-compat path.
- **No `dyn` proliferation in new traits.** Per Constitution Principle 4 + 2026 quality bar. `OneSequenceGestureRecognizer` + `PrimaryPointerGestureRecognizer` traits stay generic-bounded; concrete recognizers monomorphize.
