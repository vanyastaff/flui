# Compositing + A11y Repair — `flui-layer` × `flui-semantics`

## Summary

Single comprehensive PR repairing the FLUI compositing + accessibility layer. Closes 25 findings from [`docs/research/2026-05-22-flui-layer-semantics-audit.md`](../research/2026-05-22-flui-layer-semantics-audit.md) in atomic-commit-per-unit shape matching [PR #84](https://github.com/vanyastaff/flui/pull/84) precedent. Three architectural cleanups dominate the CRITICAL tier: (a) **Layer lifecycle protocol phases 1+2** — Flutter's [`layer.dart:144-340,372,447`](../../.flutter/flutter-master/packages/flutter/lib/src/rendering/layer.dart) ships `_refCount` + `dispose()` + `_engineLayer` + `_needsAddToScene`; FLUI ships zero of those. This PR adds `LayerNode::disposed: AtomicBool` + `impl Drop` + `debug_assert!` use-after-disposal guards (phase 1, mirrors PR #84's `ChangeNotifier::dispose` pattern at [`flui-foundation/src/notifier.rs`](../../crates/flui-foundation/src/notifier.rs)) and `LayerNode::needs_add_to_scene: AtomicBool` + `LayerTree::mark_needs_add_to_scene` + `update_subtree_needs_add_to_scene` dirty-bit propagation (phase 2, mirrors Flutter [`layer.dart:495-521`](../../.flutter/flutter-master/packages/flutter/lib/src/rendering/layer.dart) `updateSubtreeNeedsAddToScene`). Phase 3 (`engine_layer: Option<EngineLayerHandle>` cache) is explicitly DEFERRED to flui-engine materialization cycle — handle type doesn't exist yet, blocking premature work; (b) **`Layer` enum variant boxing** — 19-variant enum carries `#[allow(clippy::large_enum_variant)]` discount with `PictureLayer`/`CanvasLayer`/`ClipPathLayer`/`PerformanceOverlayLayer` heavy variants forcing every `LayerNode::layer` slot to the worst-case footprint (~360 bytes vs 16 bytes lightest). Constitution Principle 6 violation in spirit. Mirror the existing `announce_callback` registration pattern.

Adjacent HIGH-tier cleanups stack: **Slab-tree hygiene pair** — both `LayerTree::remove` and `SemanticsTree::remove` document "does NOT remove children. Caller must handle tree cleanup" — footgun-shaped API that orphans children on slab; both `LayerTree::add_child` and `SemanticsTree::add_child` do NOT detach from previous parent, silently allowing dual-parent state corrupting top-down tree walks. Cascade-by-default `remove` + auto-detach `add_child` matching Flutter [`layer.dart:1098-1149,1185-1216`](../../.flutter/flutter-master/packages/flutter/lib/src/rendering/layer.dart) `ContainerLayer.append` (asserts `child._parent == null`) + `ContainerLayer.remove` (clears sibling pointers, cascades via `LayerHandle._unref()`). Lands as SUB-UNIT inside the layer PR (not separate flui-types PR) — atomic-per-unit ownership keeps newtype + sole consumer in one commit. **`SemanticsConfiguration::absorb` 4-point Flutter drift** — first-wins for label/value/hint where Flutter [`semantics.dart:6837-6862`](../../.flutter/flutter-master/packages/flutter/lib/src/semantics/semantics.dart) concatenates label + hint via `_concatAttributedString`; no `blocks_user_actions` filter (Flutter [`semantics.dart:6796-6804`](../../.flutter/flutter-master/packages/flutter/lib/src/semantics/semantics.dart) absorbs only un-blocked actions when child blocks); no role absorption (Flutter [`semantics.dart:6852-6854`](../../.flutter/flutter-master/packages/flutter/lib/src/semantics/semantics.dart)); no heading-level merge. ## Problem Frame

The audit at [`docs/research/2026-05-22-flui-layer-semantics-audit.md`](../research/2026-05-22-flui-layer-semantics-audit.md) (25 findings, 3 CRITICAL / 5 HIGH / 10 MEDIUM / 7 LOW across 46 source files + ~15.6K LOC across flui-layer 9,796 LOC + flui-semantics 5,775 LOC) surfaces four compounding load-bearing rot zones:

1. **Layer lifecycle protocol is fundamentally absent.** Flutter's `Layer` ships `_refCount: int` + `LayerHandle<T>` ref-counting + `void dispose() @mustCallSuper` + `_engineLayer: ui.EngineLayer?` retained GPU handle + `_needsAddToScene: bool` dirty bit + `updateSubtreeNeedsAddToScene()` propagation across [`layer.dart:144-340,447,372,495-521,783-822`](../../.flutter/flutter-master/packages/flutter/lib/src/rendering/layer.dart) — 4 distinct lifecycle mechanisms with `@mustCallSuper` and `@protected` annotations enforcing the contract. FLUI's `LayerNode` (`tree/layer_tree.rs:24-43`) has fields `parent / children / layer / needs_compositing / offset / element_id` with **no `disposed` flag, no `_needsAddToScene` dirty bit, no `_engineLayer` cache, no `LayerHandle` ref-counted wrapper, no `Drop` impl**. The crate was scaffolded as a *data layer* (typed enum of layer descriptions sitting in a tree) without modeling the *resource layer* (GPU-side allocations). The split between "layer as data" and "layer as resource holder" got dropped on the floor. Cost: (a) **no retained rendering** — every frame re-encodes the entire scene; `PictureLayer`'s caching benefit (replay cached commands without re-recording, `picture.rs:54-59`) is wasted because the engine never sees a cached `ui.EngineLayer` handle to skip re-tessellation; (b) **no mark-dirty propagation** — Flutter's "child dirty → parent dirty → root dirty" chain via `markNeedsAddToScene` doesn't exist; subtree-level repaint optimization requires this; only state available today is `Scene::has_content()` (binary); (c) **no double-dispose protection** — slab-removed layer can be re-inserted with the same `LayerId` (slab reuses indices); without `disposed: AtomicBool`, a stale `LayerId` from a prior frame could navigate to a freshly-inserted layer with no panic — silent corruption; (d) **resource leaks under unsafe-sister-crate** scenarios — when `flui-engine` materializes (currently disabled), GPU buffer / texture handles stored in `LayerNode::layer` won't be released on tree mutation.

2. **The only platform routing path for semantics events is a 2-line `// TODO` stub.** `SemanticsService::send_event` (`crates/flui-semantics/src/binding.rs:407-412`):
   ```rust
   #[allow(clippy::needless_pass_by_value)] // Will be consumed when routed to platform API
   pub fn send_event(event: SemanticsEvent) {
       tracing::debug!(event = ?event, "SemanticsService::send_event");
       // TODO: Route to platform accessibility API
   }
   ```
   The companion `SemanticsService::announce` (binding.rs:386-405) correctly routes through `SemanticsBinding::announce` → `announce_callback`. The `send_event` sibling was scaffolded but its `event_callback` registration was never added. `SemanticsService::tooltip` (binding.rs:414-418) constructs a `SemanticsEvent::tooltip` then calls the broken `send_event` — silently no-ops. Future Flutter-aligned events `SemanticsEvent::scroll_completed` / `tap` / `longPress` / `liveRegionChanged` are all blocked behind this stub. Constitution Principle 6 in spirit ("No `unwrap()`/`println!`/`dbg!`/`unimplemented!()`/`todo!()` in production paths" — `// TODO` comment in lieu of impl is the same shape).

3. **Slab-tree mutation APIs are mutually-reinforcing footguns across both crates.** `LayerTree::remove` (`tree/layer_tree.rs:323-330`) and `SemanticsTree::remove` (`tree.rs:170-177`) both explicitly document **"This does NOT remove children. Caller must handle tree cleanup"** — non-cascade leaves orphan children in the slab; if a fresh insert reuses the slab index, the orphan children's `.parent` field points to the wrong node — silent tree corruption. Flutter's `ContainerLayer.remove` (`layer.dart:1185-1216`) clears sibling pointers + relies on `LayerHandle._unref()` cascade — FLUI has no `LayerHandle`, so the cascade Flutter gets for free requires explicit cascading. `LayerTree::add_child` (`layer_tree.rs:343-353`) and `SemanticsTree::add_child` (`tree.rs:190-200`) do NOT detach the child from previous parent — if child was previously under `old_parent`, the old_parent's `children: Vec<LayerId>` still contains `child_id`. Walking top-down via `old_parent.children` incorrectly recurses into the moved child — duplicate visits, double-counting. Flutter's `ContainerLayer.append` (`layer.dart:1098-1149`) asserts `child._parent == null` — explicit contract that re-parenting requires explicit detach first. Two footguns × two trees × Slab-index-reuse semantics = silent corruption surface. Plus `LayerNode::add_child` uses `Vec::push` without duplicate check (calling `add_child(p, c)` twice silently appends `c` twice); `SemanticsNode::add_child` (`node.rs:127-130`) HAS containment check — asymmetric protection.

4. **Flutter `SemanticsConfiguration::absorb` divergence at 4 points compounds into screen-reader truncation.** Flutter [`semantics.dart:6790-6862`](../../.flutter/flutter-master/packages/flutter/lib/src/semantics/semantics.dart): (a) `_attributedLabel = _concatAttributedString(this, child)` — labels CONCATENATE (text-direction-aware); (b) `if (child.isBlockingUserActions) { only unblocked actions absorbed }` — `_kUnblockedUserActions` mask filter; (c) `if (_role == SemanticsRole.none) { _role = child._role }` — role merger; (d) `_attributedHint = _concatAttributedString(...)` — hint CONCATENATES too. FLUI's `absorb` (`configuration.rs:820-854`): first-wins for label/value/hint, always copies all actions ignoring `blocks_user_actions`, no role field at all, no heading-level support. For a button with label "Submit" containing a child label "loading state" the merged accessibility output should read "Submit loading state" per Flutter; FLUI returns "Submit". Combined cost: screen readers see truncated content for any merged-semantics widget; blocked-user-actions flag is stored but never consulted (`configuration.rs:53-54`); the 28-variant `SemanticsRole` enum (`role.rs:32-141`) is exposed in the prelude but cannot be assigned to any `SemanticsConfiguration` — dead public surface that should earn its keep by becoming an absorbing field.

The cost is invisible because no test exercises the broken paths. FLUI advertises Flutter widget-tree parity + retained rendering + platform a11y integration, but **no `Layer` instance disposes cleanly**, **no platform a11y event reaches the OS** (other than announce), **no subtree-level repaint is possible without `_needsAddToScene`**, **no merged-semantics widget has Flutter-faithful absorption**. Slab-index reuse + dual-parent state both produce silent corruption with no test coverage. Repeat reviews of the same broken surface across distinct PRs would be the pattern the user explicitly forbids per [memory `no-quick-wins-vanyastaff`](../../../../.claude/projects/C--Users-vanya-RustroverProjects-flui/memory/no-quick-wins-vanyastaff.md) — bundle into one repair pass matching PR #84 multi-unit precedent + PRs #85-#98 atomic-commit cadence.

## Actors

- **A1. Layer-tree consumer (`flui-rendering` repaint-boundary path, currently disabled).** Constructs a `Scene`, calls `Scene::builder()` to mutate via `SceneBuilder::push_*` / `add_*` / `pop`, then `build()`s. Expects every `LayerNode` to dispose cleanly on `LayerTree::clear()` or `SceneBuilder` drop; expects re-paint of unchanged subtree to skip GPU re-encoding via `_engineLayer` retention; expects `remove(parent_id)` to cascade through children safely; expects `add_child(p, c)` to detach `c` from its prior parent.
- **A2. Semantics-tree consumer (`flui-rendering` / `flui-view`, when materialized).** Inserts `SemanticsNode`s with `SemanticsConfiguration` payloads, calls `SemanticsTree::add_child(parent, child)`, expects detach-from-prev-parent + dedup-on-add. Calls `SemanticsOwner::flush()` per frame, expects no allocator churn for the dirty Vec. Calls `SemanticsService::send_event(SemanticsEvent::tooltip("Submit"))`, expects the OS a11y service to receive the event.
- **A3. Scene-builder pipeline (`flui-engine` consumer, currently disabled).** Receives `Scene` from layer tree, walks the tree variants, encodes a GPU command stream. Expects `Layer` enum to be reasonably-sized (today every node carries 360+ bytes worst-case for `PictureLayer` payload). Expects `LayerNode::engine_layer: Option<EngineLayerHandle>` slot for retained-rendering optimization — DEFERRED to phase 3, blocked on flui-engine handle type materialization.
- **A4. Layer host / repaint-boundary owner (`RenderRepaintBoundary` analog, deferred).** Wraps a subtree behind a `LayerHandle<T>` ref-counted handle that auto-disposes the contained Layer when the boundary widget's subtree is unmounted. FLUI has no `LayerHandle` — only ad-hoc `LayerTree::remove` calls. Cost: no auto-dispose on subtree unmount → leaked engine resources when flui-engine materializes. Phase 1 (this PR) establishes the `disposed: AtomicBool` foundation; ref-counted handle wrapper is post-phase-2 work.
- **A5. A11y platform sink (`flui-platform` future consumer).** Registers `SemanticsBinding::set_event_callback(Arc<dyn Fn(&SemanticsEvent)>)` once during platform startup; receives `SemanticsService::send_event` dispatches; translates to native API (UIA on Windows, NSAccessibility on macOS, AT-SPI on Linux). Today: callback hook doesn't exist on `SemanticsBinding`; the routing pipe is broken at the FLUI side.
- **A6. Animation pipeline `disable_animations` check (60fps hot path).** Calls `SemanticsBinding::instance().disable_animations()` per-frame to gate animation work. Today: takes `RwLock<AccessibilityFeatures>::read()` for a single bool read. Expects: `AccessibilityFeatures` packed into `AtomicU8` bitflags + lockless `load(Acquire)`. *Rust Atomics and Locks* (Gjengset) ch. 3 — scalar reads should be atomic, not lock-mediated.

## Key Flows

- **F1. Layer dispose (lifecycle phase 1).** A1 mutates a Scene via `SceneBuilder`. `Scene::layer_tree.remove(id)` drops the `LayerNode` from slab — `impl Drop for LayerNode` fires, sets `disposed: AtomicBool` to `true` via `Release` store, optionally releases engine_layer cache (phase 3). Subsequent `LayerNode::layer_mut()` / `set_*` / `add_child` mutation paths `debug_assert!(!self.disposed.load(Acquire), "LayerNode used after disposal")`. Release builds emit `tracing::warn!` + no-op (Flutter parity: assertion in debug, warn in release). Mirrors PR #84 [`flui-foundation/src/notifier.rs`](../../crates/flui-foundation/src/notifier.rs) `ChangeNotifier::dispose` + `is_disposed` pattern. Flutter parity: [`layer.dart:319-340`](../../.flutter/flutter-master/packages/flutter/lib/src/rendering/layer.dart) `void dispose() @mustCallSuper` + `_debugDisposed` flag. Covered by **R1, R2, R3, AE1, AE2**.

- **F2. Retained rendering with `engine_layer` cache (Flutter behavior, FLUI phase 3 DEFERRED).** When materialized: A1's Scene builder pushes a `PictureLayer`, the engine encodes the picture to a `EngineLayerHandle` and stores it in `LayerNode::engine_layer`. Next frame: if the picture is unchanged AND `needs_add_to_scene == false`, the engine REUSES the cached handle without re-encoding. When `set_picture(new_picture)` is called, `engine_layer` slot is cleared via `engine_layer = None` invalidation. On `LayerNode::drop`, the handle is released. **Status: blocked on flui-engine handle type; this PR adds the field-shape foundation via phases 1+2 but does NOT add the `engine_layer` field. See Scope Boundary §1.** Flutter parity: [`layer.dart:447-481`](../../.flutter/flutter-master/packages/flutter/lib/src/rendering/layer.dart) `_engineLayer` field + `set engineLayer` invalidator.

- **F3. `needs_add_to_scene` dirty bit propagation (lifecycle phase 2).** A1 mutates a `LayerNode::layer_mut()` (or calls a `Layer::set_*` setter). The mutation path calls `LayerTree::mark_needs_add_to_scene(id)` which sets `LayerNode::needs_add_to_scene: AtomicBool` to true and walks ancestors via `TreeNav::ancestors` setting each one's flag to true. At compositing time, the engine calls `LayerTree::update_subtree_needs_add_to_scene(root)` matching Flutter [`layer.dart:495-521`](../../.flutter/flutter-master/packages/flutter/lib/src/rendering/layer.dart) `updateSubtreeNeedsAddToScene`: post-order walk computes per-subtree dirty bit (parent dirty if any child dirty OR own flag set). After scene encoding, dirty bits are cleared. Covered by **R4, R5, AE3**.

- **F4. `SemanticsService::send_event` platform routing.** A5 registers `SemanticsBinding::set_event_callback(arc_closure)` at platform startup. A2 calls `SemanticsService::send_event(SemanticsEvent::tooltip("Save"))`. `SemanticsService::send_event` checks `SemanticsBinding::is_initialized()`; if yes, calls `SemanticsBinding::instance().dispatch_event(&event)` which CLONES the `Arc<dyn Fn>` out of the `RwLock<Option<...>>::read()`, RELEASES the lock, then invokes the callback (clone-and-release per Gjengset; prevents lock-held-during-callback class of bugs). A5's closure forwards to the native a11y API. Mirrors the existing `SemanticsBinding::announce` / `set_announce_callback` / `dispatch_announce` triad at [`binding.rs:158-162,238-243,261-270`](../../crates/flui-semantics/src/binding.rs). Covered by **R6, R7, AE4, AE5**.

- **F5. Layer-tree cascade detach.** A1 calls `LayerTree::remove(parent_id)`. The cascading variant (the new default behavior): post-order walks `parent_id`'s children, recursively calling `remove(child_id)` on each, then removes `parent_id` from its own parent's children Vec, then drops the `LayerNode` from slab. Each LayerNode drop triggers F1. Recursion depth bounded by `TreeNav::MAX_DEPTH = 32`. The `remove_shallow` API (renamed from old `remove`) remains for callers that explicitly want non-cascading removal (e.g., reparenting workflows that re-insert the children elsewhere). Mirrors Flutter [`layer.dart:1185-1216`](../../.flutter/flutter-master/packages/flutter/lib/src/rendering/layer.dart) `ContainerLayer.remove` + `LayerHandle._unref()` cascade. Covered by **R8, AE6**.

- **F6. Dual-parent prevention on `add_child`.** A1 calls `LayerTree::add_child(new_parent_id, child_id)`. The new behavior: read `child.parent()`; if Some(prev_parent) and prev_parent != new_parent_id, find prev_parent_node and call `prev_parent_node.remove_child(child_id)` first. Then update new_parent's children + child's parent. The `LayerNode::add_child` impl also adopts the containment check that `SemanticsNode::add_child` already has — `if !self.children.contains(&child) { self.children.push(child); }`. Asymmetric protection (Layer Vec lacks check, Semantics Vec has it) becomes uniform. Mirrors Flutter [`layer.dart:1098-1149`](../../.flutter/flutter-master/packages/flutter/lib/src/rendering/layer.dart) `ContainerLayer.append` (asserts `child._parent == null`). Covered by **R9, AE7**.

- **F7. Flutter-faithful `SemanticsConfiguration::absorb` (label + hint concatenation, blocks-actions filter, role merge).** During semantics tree assembly: child node merges into parent via `parent.config.absorb(&child.config)`. The new absorb: (a) label = `_concat_attributed_string(self.label, self.text_direction, child.label, child.text_direction)` if both have content (text-direction-aware); else first-wins; (b) hint = same as label (Flutter line 6852); (c) value = first-wins (Flutter line 6841-6843: only assigns if self is empty); (d) actions: if `child.blocks_user_actions == true`, only absorb actions whose bit is in `UNBLOCKED_USER_ACTIONS_MASK` (Flutter `_kUnblockedUserActions`); else absorb all; (e) role: if `self.role == SemanticsRole::None`, assign `self.role = child.role`. Covered by **R10, R11, R12, AE8, AE9, AE10**.

- **F8. Semantics + Layer enum unified type baseline.** A1 reads `LayerNode::layer()`; pattern matches on `Layer::Picture(_)` where the variant carries `Box<PictureLayer>` after boxing. The `as_picture(&self) -> Option<&PictureLayer>` accessor (auto-derived via `gen_layer_accessors!` macro at `layer/dispatch.rs:88`) dereferences the Box automatically via `Deref`. `From<PictureLayer> for Layer` impl now boxes via `Layer::Picture(Box::new(picture))`. Other boxed variants: `Canvas`, `ClipPath`, `PerformanceOverlay`. Lightweight variants (Opacity / Offset / Leader / Follower / Transform / ClipRect / ClipRRect / ClipSuperellipse / ColorFilter / ImageFilter / Texture / BackdropFilter / PlatformView / ShaderMask / AnnotatedRegion) stay inline. `#[allow(clippy::large_enum_variant)]` removed. Covered by **R13, AE11**.

## Requirements

**Zero-dependency deletions (Wave 1 — no consumer impact, foundational cleanup):**

- **R0.** Delete zero-consumer scaffolding from flui-layer: `LayerNode::needs_compositing: bool` field + `set_needs_compositing` setter + `needs_compositing()` getter (`tree/layer_tree.rs:36, 124-134`, stale cache shadows `Layer::needs_compositing()` enum-method; 0 production callers verified via grep); `LayerNode::set_offset` + `set_element_id` mutator setters (`layer_tree.rs:144-146,156-158`, 0 production callers; builder pattern via `with_offset` / `with_element_id` is the canonical mutation path); `Scene::layer_tree_mut` (`scene.rs:261-263`, 0 production callers; `Scene::builder()` is canonical mutation entry); `LinkRegistry::leader_for_follower` / `links()` / `leaders()` iterators (`link_registry.rs:251-265`, 0 production callers); `LinkRegistry::rebuild_follower_lists` (`link_registry.rs:314-326`, 0 production callers; deserialization recovery path that doesn't exist); `SceneCompositor::release` + `clear_retained` (`compositor/retained.rs:78-86`, 0 production callers; speculative retain-flow); `LayerTree::iter_mut` (`layer_tree.rs:531-535`) kept with `#[allow(dead_code)]` (renderer convenience for future). Total scheduler-side: ~150 LOC deletion, zero workspace consumers confirmed via Appendix A grep receipts in audit.

**Type baseline + Principle 6 (Wave 2 — blocks Wave 3 lifecycle pattern adoption):**

- **R1.** Introduce `flui_types::painting::Alignment` newtype matching Flutter [`painting/alignment.dart:275-310`](../../.flutter/flutter-master/packages/flutter/lib/src/painting/alignment.dart): two-component `(x: f32, y: f32)` range `-1..=1` (visual coordinates), const constructors `TOP_LEFT = (-1, -1)`, `TOP_CENTER = (0, -1)`, `TOP_RIGHT = (1, -1)`, `CENTER_LEFT = (-1, 0)`, `CENTER = (0, 0)`, `CENTER_RIGHT = (1, 0)`, `BOTTOM_LEFT = (-1, 1)`, `BOTTOM_CENTER = (0, 1)`, `BOTTOM_RIGHT = (1, 1)`. Lerp helper `Alignment::lerp(a: Alignment, b: Alignment, t: f32) -> Alignment` per Flutter line 222. `align_within(rect: Rect<Pixels>) -> Offset<Pixels>` helper produces the pixel-coordinate within a rect at this alignment. Lands as a SUB-UNIT inside this PR (not separate flui-types PR — bundles atomic-per-unit with the FollowerLayer consumer migration). `#[non_exhaustive]` future-compat. `#[must_use]` on lerp + align_within.
