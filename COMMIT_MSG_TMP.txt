fix(rendering): PR-A2 Codex P2 — add_node_needing_compositing_bits_update sets flag on enqueue

Codex #3294562493 flagged a silent-loss footgun: pre-fix
add_node_needing_compositing_bits_update pushed to the dirty queue
without setting NEEDS_COMPOSITING_BITS_UPDATE on the node. If a caller
enqueued without independently calling
mark_needs_compositing_bits_update, run_compositing's per-entry
short-circuit (early-return when the flag is false) would silently
drop the queued work AND clear the queue at end of phase.

Fix: add_node_needing_compositing_bits_update now sets the flag via
node.mark_needs_compositing_bits_update() before the queue push. The
new invariant — queue entry ⇒ flag set — eliminates the silent-loss
path. Callers that want flag-set without queue membership reach for
RenderNode::mark_needs_compositing_bits_update directly.

Test updates:
* Renamed u34_run_compositing_short_circuits_when_flag_not_set →
  u34_run_compositing_short_circuits_when_flag_cleared_after_enqueue
  (the test now clears the flag post-enqueue to simulate the
  parent-cleared-me-mid-walk case).
* Added u34_add_node_needing_compositing_bits_update_sets_flag_on_enqueue
  regression test asserting the invariant.

All 7 U34 tests pass; full flui-rendering lib + integration green.
